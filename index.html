一、关系数据库概述

数据模型

  • 层次模型（树状）
  • 网状模型
  • 关系模型（二维表格）

数据类型

  1. INT：整型 4
  2. BIGINT: 长整型 8 常用
  3. REAL: 浮点型 4
  4. DOUBLE: 浮点型 8
  5. DECIMAL(M, N): 高精度小数 共20位，其中小数10位
  6. CHAR: 定长字符串
  7. VARCHAR: 变长字符串
  8. BOOLEAN
  9. DATE
  10. TIME
  11. DATETIME

SQL (Structured Query Language)

  1. Oracle把自己扩展的SQL称为 PL/SQL
  2. DDL(Data Definition Language)：创建、删除、修改表结构 数据库管理员
  3. DML(Data Manipulation Language): 添加、删除、更新数据的能力 应用程序对数据库的操作
  4. DQL(Data Query Language): 用户查询数据 用户

CRUD：Create、Retrieve、Update、Delete

良好习惯

SQL关键字总是大写，表名和列名均使用小写。

二、关系模型

  1. 记录：表的每一行
  2. 字段：表的每一列（字段应该避免允许为NULL）

主键

  1. 定义：能够唯一标识一条记录的某个字段

  2. 基本原则：不使用任何业务相关的字段作为主键，一般使用id

联合主键（不常用）

通过多个字段唯一标识记录，允许一列有重复，只要不是所有主键列都重复即可

外键（也可以不设置约束，仅依靠应用程序的逻辑来保证）

  1. 一对多关系（一个班级对应多名学生）

    在 多 的表里首先添加一个字段

  1. 多对多关系（一个老师对应多个班级，一个班级对应多个老师）

    创建一个中间表（建立两个一对多的关系）

  1. 一对一关系（一个表的记录对应到另一个表的唯一一个记录，一个学生对应一个电话号码）

    用于将常读取和不常读取字段分开，提高查询速度

索引

关系数据库中对某一列或多个列的值进行预排序的数据结构

值越不相同，索引效率越高

唯一索引：不宜作为主键，但具有唯一性约束

三、查询数据

准备数据

基本查询

  1. 查询所有

    **SELECT * FROM students;**

  2. 测试数据库连接

    **SELECT 1;**

条件查询

  1. 获取成绩在80分以上的

    **SELECT * FROM students WHERE score >= 80;**

  2. 获取成绩在80分以上且是“男生”

    **SELECT * FROM students WHERE score >= 80 AND gender = 'M'**

  3. 获取成绩在80分以上或者是“男生”

    **SELECT * FROM students WHERE score >= 80 OR gender = 'M'**

  4. 获取班级不为1的

    **SELECT * FROM students WHERE class_id <> 1;**

  5. 获取分数在80以下或者90以上，并且是男生

    **SELECT * FROM students WHERE (score < 80 OR score > 90) AND gender = 'M';**

  6. 获取名字带“王”的（LIKE ‘%王’）

    **SELECT * FROM students WHERE name LIKE "%王%";**

投影查询

  1. 获取指定列并将name列改为stuName

    **SELECT id, name stuName, score FROM students;**

  2. 获取指定列，且是男生

    **SELECT id, name stuName, score FROM students WHERE gender = 'M';**

排序（默认按照主键排序）

  1. 根据成绩排序（默认从低到高 ORDER BY）

    SELECT * FROM students ORDER BY score;

  2. 根据成绩排序（从高到低）

    SELECT id, name, score FROM students ORDER BY score DESC;

  3. 先根据成绩升序排列，如果成绩有相同的，再按性别降序排列排序

    SELECT id, name, gender, score FROM students ORDER BY score ASC, gender DESC;

  4. 查询一班的学生成绩，并按照倒序排列

分页查询

  1. 查询学生的成绩，分页显示，从第1条显示，每页最多显示3条(LIMIT 3 OFFSET 0;)

  2. 每页的开始OFFSET = 每页要显示的条数 * （当前页码 - 1）

从 OFFSET 条开始显示，每页有 LIMIT 条

聚合查询

  1. 查询表中共有多少行数据，以num显示

    **SELECT COUNT(*) num FROM students;**

  2. 查询有多少男生

    **SELECT COUNT(id) boys FROM students WHERE gender = 'M';**

  3. 其他聚合函数

  1. 使用AVG查询男生的平均成绩（聚合查询使用WHERE条件）

    **SELECT AVG(score) average FROM students WHERE gender = 'M';**

  2. 每页3条记录，通过聚合查询获得总页数

    **SELECT CEILING(COUNT(*) / 3) FROM students;**
分组

  1. 查询每个班级的人数

    **SELECT class_id, COUNT(*) num FROM students GROUP BY class_id;**

  2. 每个班级的平均分

    **SELECT class_id, AVG(score) ave FROM students GROUP BY class_id;**

  3. 每个班级男生女生的平均分，降序排列

多表查询(慎用)

隔离级别（从低到高）

  1. Read Uncommitted

    事务A 会读到 事务B 更新后但未提交的数据

    脏读：事务B 回滚，那么 事务A 读到的数据就是脏数据（和上一次读到的数据不一致）

  2. Read Committed

    一个事务可能会遇到不可重复读（Non Repeatable Read）的问题

    不可重复读：在事务A内，多次读同一数据，若在这个事务没结束前，事务B恰好修改了这个数据，那么事务A中，两次读取的数据可能就不一致

  3. Repeatable Read

    事务可能会遇到幻读（Phantom Read）的问题

    幻读：在一个事务中，第一次查询某条记录，没有

    但是，当试图更新这条不存在的记录时，竟然能成功

并且，再次读取同一条记录，它就神奇地出现了

  4. Serializable

    最严格的隔离级别，所有事务按照次序依次执行

  5. MySQL中，如果使用InnoDB，默认的隔离级别是Repeatable Read

异常

1. Throwable类

所有异常的超类，出现Exception代表程序有异常，可以进行处理

常用方法：

  • Throwable()； 构造方法，Throwable类对象中，存储的异常或者错误信息为null

  • Throwable(String message); Throwable类对象中，存储的异常或者错误信息为message

  • String getMessage(); 获取Throwable对象中存储的异常或者错误信息

  • String toString(); 返回当前异常或者错误的简要描述

  • void printStackTrace(); 展示错误的前因后果，【红色字体】

2. 异常处理

特征：

  • 代码中从异常发生位置开始，之后的代码都不再运行
  • 代码中有多个异常，可以使用多个catch块进行捕获操作，分门别类处理
  • 当前情况下，只能展示异常情况，后期可以将异常情况做成log日志文件
  • 异常被捕获之后代码可以正常运行

3. 抛出异常

特征：

  • 一个代码块内，有且只能抛出一个异常
  • 从throw位置开始，之后的代码不在运行
  • 代码中存在使用throw抛出异常，在方法的【声明位置】必须告知调用者这里有什么异常

4. 抛出和捕获的对比

  • 捕获之后，代码可以正常运行，要保证处理之后的异常不会再导致其他问题。
  • 抛出，可以让代码健壮性很强，到用户层面不可抛出，不友好

用户名密码错误，不能采用捕获异常。应该通过以下方式处理

5. RuntimeException

运行时异常，在代码中如果出现，不需要代码中强制进行捕获或者抛出处理。JVM非检查

6. 自定义异常

自定义异常类名：必须Exception结尾！！！

类

对于一类事物的统称，对当前事物的一些描述，属性描述和行为描述

交通工具类

对象

独立，唯一，特殊的个体

汽车，轮船，飞机

this关键字（调用本类的）

  1. 表示调用当前方法的类对象
  2. 当前构造方法中初始化的类对象

局部变量

未赋值不能参与除赋值之外的其他运算。

GC机制

  1. 在单位时间内，检查当前Java程序使用的内存中是否存在无主内存。
  2. 标记无主内存，多次标记的情况下，会将无主内存释放，归还内存空间。

面向对象三大特征

封装、继承、多态

封装

JavaBean规范的类封装过程

  1. 所有的成员变量全部私有化 ==> private
  2. 必须提供一个无参数构造方法
  3. 使用private修饰的成员变量提供Setter Getter方法

匿名对象（不要操作成员变量）

用途：

  • 使用匿名对象直接调用类内的成员方法
  • 匿名对象直接作为方法的参数

继承（extend）

  • 子类继承父类之后，可以使用父类的非私有化成员变量和成员方法
  • 子类不能继承得到父类的私有化内容。

super关键字（调用父类的）

  • 如果没有显式调用父类的构造方法，Java编译器会默认调用无参父类构造方法使用
  • 根据数据类型选择对应方法
  • super调用构造方法，必须在当前构造方法的第一行

方法重载

一个类中，名字相同，参数个数、类型不同的方法

方法重写

父类的方法不适用于子类情况，子类可以重写父类的方法

返回值类型，方法名和形式参数列表和父类完全一致

强制子类重写方法（abstract）

  • abstract修饰的方法没有方法体

  • abstract修饰的方法必须定义在abstract修饰的类内或者interface接口内

  • 一个普通类继承了一个abstract类，必须实现在abstract类内的所有abstract方法

  • 一个abstract类继承了另一个abstract类，选择实现其中的abstract方法

  • abstract修饰的类内允许普通方法

  • abstract类不能有自己的类对象

final关键字

  • 修饰成员变量

    • 定义时必须初始化
    • 赋值之后无法修改

  • 修饰成员方法

    • 不能被子类重写

  • 修饰类

    • 没有子类，不能被继承

abstract修饰的类不能被 final 修饰

static关键字（通过类名调用）

一、修饰成员变量

1. 修饰成员变量的特征

  • 公共区域存放
  1. 都可以使用
  2. 和对象无关
  3. 一处修改，处处受到影响。

2. 注意事项

  • 使用类名调用静态成员变量
  • 静态成员变量应该通过静态方法访问

3. 静态成员变量和对象无关

  • 内存角度
    • 静态成员变量 —– 内存的数据区
    • 类对象 —- 内存的堆区
  • 生命周期
    • 静态成员变量 —- 从程序运行开始就已存在，随着(.class) 字节码文件的加载过程中，直接定义在内存的数据区 —- 从程序开始，到程序结束
    • 类对象 —- 是在代码的运行过程中，有可能被创建的 —- 从创建开始，而且随时有可能被JVM的
      GC销毁

二、修饰成员方法

1. 注意事项

  • 使用静态方式调用，通过类名调用
  • 静态成员方法中不能使用非静态成员，可以使用其他静态成员
  • 静态成员方法中不能使用this关键字
  • 静态成员方法中可以通过new 构造方法创建对象

2. 特征

  • 静态成员方法是随着.class字节码文件的加载而直接定义在内存的【方法区】
  • 不能使用非静态成员，不能使用this关键字【没有对象】
  • 类内成员变量和成员方法都不可以使用，但是不影响使用外来数据

3. 用途

  • 静态成员方法通常用于工具类的封装使用

三、类变量、类方法

  • 类变量 ==> 静态成员变量
  • 类方法 ==> 静态成员方法
  • 类成员 ==> 静态成员变量和静态成员方法

类方法可以使用当前类内的静态成员变量，但是不允许使用非静态成员变量

四、静态代码块

1. 构造代码块

只要调用构造方法，一定会执行对应的构造代码块（初始化当前类的所有类对象）

2. 静态代码块

只要类文件加载，静态代码块中所有内容全部执行，有且只执行一次

特征：
  • 不能使用this关键字，不能使用类内的非静态成员
  • 可以使用类内的其他静态成员
  • 定义的变量都是局部变量

3. 局部代码块

提高效率，解决内存，让JVM回收内存的效率提升。
