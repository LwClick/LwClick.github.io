ArrayList性能问题

1. 增删慢

  • 增加慢：增加元素有可能出现调用grow方法，grow需要进行数组的扩容操作，大量的移动和拷贝
  • 删除慢：按照ArrayList可变长数组要求，删除元素之后，之后的内容都需要整体向前移动。

2. 内存

  • 最小单元 字节
  • 给内存进行编号，标记每一个字节的内存，通过内存编号访问对应的内存，查询效率高
  • 一个字节对应一个地址，从0开始编号到34359738367 (32G内存)
  • 引入16进制 0x0 ~ 0x7 FFFF FFFF

3. null

  • null ==> 0x0 内存中编号为0的地址
  • 该地址受到系统保护，任何程序读取，写入0x0地址，系统直接杀死程序

LinkedList

1. 特征：

  • 底层是一个双向链表，增删快，查询很慢很慢
  • 自行车链子： 维修很方便，前断，后断，链接搞定！！！ 找到损坏的位置，需要一个一个来
  • 非连续空间存储
