一、关系数据库概述

数据模型

  • 层次模型（树状）
  • 网状模型
  • 关系模型（二维表格）

数据类型

  1. INT：整型 4
  2. BIGINT: 长整型 8 常用
  3. REAL: 浮点型 4
  4. DOUBLE: 浮点型 8
  5. DECIMAL(M, N): 高精度小数 共20位，其中小数10位
  6. CHAR: 定长字符串
  7. VARCHAR: 变长字符串
  8. BOOLEAN
  9. DATE
  10. TIME
  11. DATETIME

SQL (Structured Query Language)

  1. Oracle把自己扩展的SQL称为 PL/SQL
  2. DDL(Data Definition Language)：创建、删除、修改表结构 数据库管理员
  3. DML(Data Manipulation Language): 添加、删除、更新数据的能力 应用程序对数据库的操作
  4. DQL(Data Query Language): 用户查询数据 用户

CRUD：Create、Retrieve、Update、Delete

良好习惯

SQL关键字总是大写，表名和列名均使用小写。

二、关系模型

  1. 记录：表的每一行
  2. 字段：表的每一列（字段应该避免允许为NULL）

主键

  1. 定义：能够唯一标识一条记录的某个字段

  2. 基本原则：不使用任何业务相关的字段作为主键，一般使用id

联合主键（不常用）

通过多个字段唯一标识记录，允许一列有重复，只要不是所有主键列都重复即可

外键（也可以不设置约束，仅依靠应用程序的逻辑来保证）

  1. 一对多关系（一个班级对应多名学生）

    在 多 的表里首先添加一个字段

  1. 多对多关系（一个老师对应多个班级，一个班级对应多个老师）

    创建一个中间表（建立两个一对多的关系）

  1. 一对一关系（一个表的记录对应到另一个表的唯一一个记录，一个学生对应一个电话号码）

    用于将常读取和不常读取字段分开，提高查询速度

索引

关系数据库中对某一列或多个列的值进行预排序的数据结构

值越不相同，索引效率越高

唯一索引：不宜作为主键，但具有唯一性约束

三、查询数据

准备数据

基本查询

  1. 查询所有

    **SELECT * FROM students;**

  2. 测试数据库连接

    **SELECT 1;**

条件查询

  1. 获取成绩在80分以上的

    **SELECT * FROM students WHERE score >= 80;**

  2. 获取成绩在80分以上且是“男生”

    **SELECT * FROM students WHERE score >= 80 AND gender = 'M'**

  3. 获取成绩在80分以上或者是“男生”

    **SELECT * FROM students WHERE score >= 80 OR gender = 'M'**

  4. 获取班级不为1的

    **SELECT * FROM students WHERE class_id <> 1;**

  5. 获取分数在80以下或者90以上，并且是男生

    **SELECT * FROM students WHERE (score < 80 OR score > 90) AND gender = 'M';**

  6. 获取名字带“王”的（LIKE ‘%王’）

    **SELECT * FROM students WHERE name LIKE "%王%";**

投影查询

  1. 获取指定列并将name列改为stuName

    **SELECT id, name stuName, score FROM students;**

  2. 获取指定列，且是男生

    **SELECT id, name stuName, score FROM students WHERE gender = 'M';**

排序（默认按照主键排序）

  1. 根据成绩排序（默认从低到高 ORDER BY）

    SELECT * FROM students ORDER BY score;

  2. 根据成绩排序（从高到低）

    SELECT id, name, score FROM students ORDER BY score DESC;

  3. 先根据成绩升序排列，如果成绩有相同的，再按性别降序排列排序

    SELECT id, name, gender, score FROM students ORDER BY score ASC, gender DESC;

  4. 查询一班的学生成绩，并按照倒序排列

分页查询

  1. 查询学生的成绩，分页显示，从第1条显示，每页最多显示3条(LIMIT 3 OFFSET 0;)

  2. 每页的开始OFFSET = 每页要显示的条数 * （当前页码 - 1）

从 OFFSET 条开始显示，每页有 LIMIT 条

聚合查询

  1. 查询表中共有多少行数据，以num显示

    **SELECT COUNT(*) num FROM students;**

  2. 查询有多少男生

    **SELECT COUNT(id) boys FROM students WHERE gender = 'M';**

  3. 其他聚合函数

  1. 使用AVG查询男生的平均成绩（聚合查询使用WHERE条件）

    **SELECT AVG(score) average FROM students WHERE gender = 'M';**

  2. 每页3条记录，通过聚合查询获得总页数

    **SELECT CEILING(COUNT(*) / 3) FROM students;**
分组

  1. 查询每个班级的人数

    **SELECT class_id, COUNT(*) num FROM students GROUP BY class_id;**

  2. 每个班级的平均分

    **SELECT class_id, AVG(score) ave FROM students GROUP BY class_id;**

  3. 每个班级男生女生的平均分，降序排列

多表查询(慎用)

  1. 查询两个表中的属性（笛卡尔积）

连接查询

先确定一个主表作为结果集，然后，把其他表的行有选择性地“连接”在主表结果集上

1.内连接（INNER JOIN） — 只返回同时存在于两张表的行数据

查询所有学生，同时返回班级名称

2.外连接（OUTER JOIN）

六、事务管理

事务

  1. 数据库事务：把多条语句作为一个整体进行操作的功能

  2. ACID特性：

    1. A：Atomic，原子性，要么全部执行，要么全部不执行；
    2. C：Consistent，一致性，事务完成后，所有数据的状态都是一致的
    3. I：Isolation，隔离性，如果有多个事务并发执行，每个事务作出的修改必须与其他事务隔离
    4. D：Duration，持久性，即事务完成后，对数据库数据的修改被持久化存储

  3. 隐式事务：单条SQL语句，数据库系统自动将其作为一个事务执行

  4. 显式事务：

    1. 开启事务： BEGIN；

    2. 提交数据： COMMIT；

    3. 回滚事务： ROLLBACK；

    4. 设置事务的隔离级别：SET TRANSACTION ISOLATION LEVEL 级别

隔离级别（从低到高）

  1. Read Uncommitted

    事务A 会读到 事务B 更新后但未提交的数据

    脏读：事务B 回滚，那么 事务A 读到的数据就是脏数据（和上一次读到的数据不一致）

  2. Read Committed

    一个事务可能会遇到不可重复读（Non Repeatable Read）的问题

    不可重复读：在事务A内，多次读同一数据，若在这个事务没结束前，事务B恰好修改了这个数据，那么事务A中，两次读取的数据可能就不一致

  3. Repeatable Read

    事务可能会遇到幻读（Phantom Read）的问题

    幻读：在一个事务中，第一次查询某条记录，没有

    但是，当试图更新这条不存在的记录时，竟然能成功

并且，再次读取同一条记录，它就神奇地出现了

  4. Serializable

    最严格的隔离级别，所有事务按照次序依次执行

  5. MySQL中，如果使用InnoDB，默认的隔离级别是Repeatable Read
