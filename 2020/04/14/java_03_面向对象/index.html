面向对象（java笔记四）

发表于 2020-04-14 | 分类于 java | 阅读次数:
字数统计: 2.1k | 阅读时长 ≈ 8

类

对于一类事物的统称，对当前事物的一些描述，属性描述和行为描述

交通工具类

对象

独立，唯一，特殊的个体

汽车，轮船，飞机

this关键字（调用本类的）

  1. 表示调用当前方法的类对象
  2. 当前构造方法中初始化的类对象

局部变量

未赋值不能参与除赋值之外的其他运算。

GC机制

  1. 在单位时间内，检查当前Java程序使用的内存中是否存在无主内存。
  2. 标记无主内存，多次标记的情况下，会将无主内存释放，归还内存空间。

面向对象三大特征

封装、继承、多态

封装

JavaBean规范的类封装过程

  1. 所有的成员变量全部私有化 ==> private
  2. 必须提供一个无参数构造方法
  3. 使用private修饰的成员变量提供Setter Getter方法

匿名对象（不要操作成员变量）

用途：

  • 使用匿名对象直接调用类内的成员方法
  • 匿名对象直接作为方法的参数

继承（extends）

  • 子类继承父类之后，可以使用父类的非私有化成员变量和成员方法
  • 子类不能继承得到父类的私有化内容。

super关键字（调用父类的）

  • 如果没有显式调用父类的构造方法，Java编译器会默认调用无参父类构造方法使用
  • 根据数据类型选择对应方法
  • super调用构造方法，必须在当前构造方法的第一行

方法重载

一个类中，名字相同，参数个数、类型不同的方法

方法重写

父类的方法不适用于子类情况，子类可以重写父类的方法

返回值类型，方法名和形式参数列表和父类完全一致

强制子类重写方法（abstract）

  • abstract修饰的方法没有方法体

  • abstract修饰的方法必须定义在abstract修饰的类内或者interface接口内

  • 一个普通类继承了一个abstract类，必须实现在abstract类内的所有abstract方法

  • 一个abstract类继承了另一个abstract类，选择实现其中的abstract方法

  • abstract修饰的类内允许普通方法

  • abstract类不能有自己的类对象

final关键字

  • 修饰成员变量

    • 定义时必须初始化
    • 赋值之后无法修改

  • 修饰成员方法

    • 不能被子类重写

  • 修饰类

    • 没有子类，不能被继承

abstract修饰的类不能被 final 修饰

static关键字（通过类名调用）

一、修饰成员变量

1. 修饰成员变量的特征

  • 公共区域存放
  1. 都可以使用
  2. 和对象无关
  3. 一处修改，处处受到影响。

2. 注意事项

  • 使用类名调用静态成员变量
  • 静态成员变量应该通过静态方法访问

3. 静态成员变量和对象无关

  • 内存角度
    • 静态成员变量 —– 内存的数据区
    • 类对象 —- 内存的堆区
  • 生命周期
    • 静态成员变量 —- 从程序运行开始就已存在，随着(.class) 字节码文件的加载过程中，直接定义在内存的数据区 —- 从程序开始，到程序结束
    • 类对象 —- 是在代码的运行过程中，有可能被创建的 —- 从创建开始，而且随时有可能被JVM的
      GC销毁

二、修饰成员方法

1. 注意事项

  • 使用静态方式调用，通过类名调用
  • 静态成员方法中不能使用非静态成员，可以使用其他静态成员
  • 静态成员方法中不能使用this关键字
  • 静态成员方法中可以通过new 构造方法创建对象

2. 特征

  • 静态成员方法是随着.class字节码文件的加载而直接定义在内存的【方法区】
  • 不能使用非静态成员，不能使用this关键字【没有对象】
  • 类内成员变量和成员方法都不可以使用，但是不影响使用外来数据

3. 用途

  • 静态成员方法通常用于工具类的封装使用

三、类变量、类方法

  • 类变量 ==> 静态成员变量
  • 类方法 ==> 静态成员方法
  • 类成员 ==> 静态成员变量和静态成员方法

类方法可以使用当前类内的静态成员变量，但是不允许使用非静态成员变量

四、静态代码块

1. 构造代码块

只要调用构造方法，一定会执行对应的构造代码块（初始化当前类的所有类对象）

2. 静态代码块

只要类文件加载，静态代码块中所有内容全部执行，有且只执行一次

特征：
  • 不能使用this关键字，不能使用类内的非静态成员
  • 可以使用类内的其他静态成员
  • 定义的变量都是局部变量

3. 局部代码块

提高效率，解决内存，让JVM回收内存的效率提升。
